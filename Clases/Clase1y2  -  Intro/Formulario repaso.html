Formulario

Para que funcione la etiqueta "form" se necesita de 3 atributos para que funcione

  1. action: documento que se encarga de recibir los datos y procesarlos
  2. method: la forma en que sera enviada la informacion.existen dos metodos, GET y POST
  3. enctype: Como se codificaran los contenidos

Action

en este atributo indicaremos cual es el archivo que recibe y procesa los datos.debe ser de lenguaje (PHP/asp/JSP).

Metod

Forma en la que se recopilan y envian los datos,existen dos metodos comunes en el HTML

  1. GET: La informacion viajara por la barra de direcciones a continuacion del nombre del archivo
  2. POST:la informacion viajara junta a los encabezados del HTML (sera "invisible")

Por defecto (si no se indica) es GET

Ingreso de texto

Existen 3 tipos de controles generales para el ingreso de texto:

  1. cajas de texto de una sola linea (no acepta enter)
  2. cajas para el ingreso de password (contenido invisible)
  3. Cajas para contenido multilinea. Puede ser una o muchas lineas

atributo "name"

Todos los controles de formulario deben tener un nombre, que se indica con el atributo "name"
Con ese nombre despues se accede a su valor en el archivo indicado en el "action". un control sin name no se envia

control de formulario: "input"

elemento tiene multiples propositos segun el valor que tome el atributo "type". Los posibles son :

  1. type "text"
  2. type "Email"
  3. type "password"
  4. type "number"
  5. type "date" (calendario)
  6. type "file" (Subir archivo)
Atributo placeholder : texto de ayuda para el usuario
atributo required : Campo obligatorio para enviar el formulario.

"Textarea"

Es un campo con multriples lineas de texto ofrece los siguientes atributos:

  1. cols: indica la cantidad de caracteres que podra tener a lo ancho.
  2. rows: indica la cantidad de lineas de texto

Botones

Los botones disparan las acciones del formulario
Hay 3 tipos de botones

  • el que envia los datos al archivo indicado como action
  • el que vacia todo lo ingresado y resetea los campos
  • el que no hace nada y esta pensado para usarse con javascript
Todos los botones son etiquetas "input", Con distintos tipos de type
el boton debe estar dentro de "form"

Atributo "value"

representa la etiqueta del boton, la cual es normalmente mostrada por los navegadores dentro de este.



Controles de seleccion

Opciones que ofrece el programador para el usuario.El dato que nos llega al elegir una opcion se define desde el atributo "value"
Existen 3 grupos de controles de seleccion:

  • Botones de radio: Solo se puede elegir una opcion
  • Casillas de chequeo: De toda la lista de opciones, el usuario puede elegir una,todas o ninguna opcion
  • Menu desplegable: Solo se puede elegir una opcion

atributo "value"

En este caso es el valor que se enviara al enviarse el formulario

Acepta terminos y condiciones

etiqueta "label"

la etiqueta label define formalmente a cada elemento de un formulario,esta etiqueta es de mucha ayuda para generar un formulario accesible. el principal atributo de la etiqueta label es for, quen va a referenciar a label con su elemento del formulario.el valor del atributo for debe ser igual al valor del atributo id o name del elemento

menu desplegable

es el llamado combo-box, selector o menu. se necesitan dos elementos distintos:
para insertar el menu se usa la etiqueta select, que abre y cierra(pero estara vacio)
Dentro va una etiqueta option que abre y cierra por cada opcion a mostrar.
El select lleva el atributo "name" (los "option" no)
los option llevan el "value" (el select no)
Si no hay "value", se usa el texto interior.
